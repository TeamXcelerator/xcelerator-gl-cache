# xcelerator-gl-cache

Consolidated, public cache of **Gauss–Legendre quadrature node/weight
tables** used across Team Xcelerator's high-precision numerical work.

> **Author:** Ronnie Andrews, Jr. (Team Xcelerator Inc.®)  
> **License:** See `LICENSE`. Public for academic verification and
> reproduction; redistribution and commercial use restricted.

---

## What this is

A Gauss–Legendre quadrature table is uniquely determined by two
integers:

- `npts` — the number of quadrature points (nodes) on `[-1, 1]`
- `prec` — the working precision, **in bits**

The table is **project-agnostic**: a table for a given `(npts, prec)`
is the same Gauss–Legendre rule no matter which computation produced it.
That universality is why these caches can be consolidated into a single
shared repository.

Each table is generated by the Xcelerator Toolkit
(`xc-numerics::quadrature`, Newton iteration on Legendre polynomials)
and stored as a deflated zip of the canonical JSON form.

## Why it exists

High-precision GL tables are expensive to compute (Newton-on-Legendre
at thousands of digits) but cheap to store compressed. Recomputing them
on every fresh cloud run wasted hours of compute. This repo lets any
run **fetch only the tables it needs** — by deterministic URL, no clone
required — and skip the recompute. See "Fetching" below.

## Layout

Files are organized **precision-first, then bucketed by npts** in
thousands, so no directory exceeds GitHub's web-UI listing limit and
the path is deterministically computable from `(npts, prec)`:

```
gl_cache/
  prec{P}/
    npts{B}-{B+999}/
      prec{P}_npts{N}.json.zip
```

where:
- `P` = precision in bits (e.g. `3338` for HP-1000, `6660` for HP-2000)
- `N` = number of points
- `B` = `floor(N / 1000) * 1000` (the lower edge of N's thousand-bucket)

**Example:** the HP-1000 (`prec=3338`) table for `npts=4000` lives at

```
gl_cache/prec3338/npts4000-4999/prec3338_npts4000.json.zip
```

### Path formula (for tooling)

```
gl_cache/prec{P}/npts{floor(N/1000)*1000}-{floor(N/1000)*1000+999}/prec{P}_npts{N}.json.zip
```

## Fetching a single table (no clone)

This repo is public, so any table is one raw URL:

```bash
curl -sSL -O \
  https://raw.githubusercontent.com/TeamXcelerator/xcelerator-gl-cache/main/gl_cache/prec3338/npts4000-4999/prec3338_npts4000.json.zip
```

Decompress to the canonical `.json` (or let the toolkit do it
transparently on first read — it prefers `.json`, falls back to the
`.json.zip`, and writes the decompressed `.json` alongside).

## File format

Each `.json.zip` is a deflate archive containing a single entry named
`prec{P}_npts{N}.json`: a 2-element JSON array `[nodes, weights]`, where
each element is an array of `npts` decimal strings at the stated
precision. The Xcelerator Toolkit reads/writes this format directly and
structurally validates it on load (Σw = 2, Σ x·w = 0, antisymmetric
nodes) via `verify_gl_cache_dir`.
